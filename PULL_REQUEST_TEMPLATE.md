## 요구사항

### 기본

- [x] 미션3의 구현이 완료된 상태에서 진행
- [x] User 스키마 작성: id, email, nickname, imageUrls, password, createdAt, updatedAt 필드 가짐
- [x] 회원가밉 API 구현: email, nickname, passwork 입력하여 진행하고, password는 해싱하여 저장
- [x] 토큰 기반 유저 인증: 로그인에 성공하면 Access Token과 Refresh Token이 발급됨
- [x] (인증) 로그인한 유저만 상품/게시글 등록 가능
- [x] (인가) 상품/게시글을 등록한 유저한 해당 상품/게시글의 정보를 수정하거나 삭제 가능
- [x] (인증) 로그인한 유저만 상품/게시글에 댓글 등록 가능
- [x] (인가) 댓글을 등록한 유저만 해당 댓글을 수정하거나 삭제 가능
- [x] 유저가 자신의 정보를 조회하는 기능 구현
- [x] 유저가 자신의 정보를 수정하는 기능 구현
- [x] 유저가 자신의 비밀번호를 변경할 수 있는 기능 구현
- [x] 유저의 비밀번호는 리스폰스로 노출하지 않음

### 심화

- [ ] 토큰 기반 인증: Refresh Token으로 토큰을 갱신하는 기능 구현
- [ ] 유저의 상품에 '좋아요'와 '좋아요 취소'기능 구현
- [ ] 유저의 게시글에 '좋아요'와 '좋아요 취소'기능 구현
- [ ] 상품 또는 게시글을 조회할 때, 유저가 '좋아요'를 누른 항목인지 확인할 수 있도록 isLiked와 같은 불린형 필드를 리스폰스 객체에 포함시켜 리스폰스
- [ ] 유저가 '좋아요'를 표시한 상품의 목록을 조회하는 기능 구현
